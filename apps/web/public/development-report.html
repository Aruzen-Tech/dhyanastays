1. Platform Overview

Dhyana Stays is a full-stack wellness retreat booking platform built for the Indian market. It connects mindful travellers with curated retreat properties — from Himalayan yoga centres to coastal Ayurveda resorts.

The platform handles the complete booking lifecycle: discovery → quote → hold → payment → stay → review, with dedicated consoles for guests, hosts, and administrators.

🔄

Full Booking Lifecycle

7-step flow from discovery to completion with 8 booking statuses and automated transitions

👥

Three User Roles

Guest, Host, and Admin each with dedicated dashboards, APIs, and access-controlled pages

💳

Razorpay Payments

Full/deposit payment plans, webhook verification, automatic refunds, and weekly host payouts

🗺️

Map Discovery

Leaflet-based map with grid/map/split views, geo-bounding-box filtering, and marker popups

💬

Built-in Messaging

Guest↔Host and Host↔Admin conversations with unread counts and real-time polling

🌿

Wellness-Focused UX

Retreat preparation guides, wellness preferences, digital check-in/out, and property manuals

7. Security Practices

Dual-mode Authentication

Custom JWT (HS256) for development; Auth0 JWKS (RS256) for production. Automatic token refresh on 401 responses.

Role-Based Access Control

Every endpoint decorated with @Roles(). JwtAuthGuard + RolesGuard applied globally. Guests cannot access host/admin routes and vice versa.

HMAC Price Snapshots

Every price quote is HMAC-signed before being sent to the client. The backend rejects any booking whose price snapshot fails verification — prevents client-side price manipulation.

Idempotency Keys

Payments and bookings require a UUID idempotency key. DB-backed deduplication prevents double charges from network retries (24-hour window).

Razorpay Webhook Verification

All incoming Razorpay webhook events are verified using HMAC-SHA256 signature. Events with invalid signatures are rejected with 400.

Rate Limiting

100 requests per minute per IP globally (NestJS Throttler). Login endpoint has a stricter LoginThrottleGuard. Violations tracked in DB for admin review.

Production ENV Validation

Joi schema blocks startup if stub providers (Razorpay, email, SMS, storage) are used in NODE_ENV=production. Prevents accidental misconfiguration.

Security Headers

Helmet middleware (NestJS) adds X-Frame-Options, NOSNIFF, HSTS, and CSP headers. Next.js adds X-Frame-Options: DENY and Referrer-Policy on all routes.

Audit Logging

Every create/update/delete writes an immutable audit entry (actor, action, target, diff). Full audit log accessible to admin with filtering and pagination.

Financial Ledger

Double-entry ledger records every money movement: payment capture, refund, payout. Cannot be modified — provides a full financial audit trail.

Input Validation

class-validator DTOs on all POST/PATCH endpoints. Whitelist mode strips unknown fields. Custom validators for amounts, dates, and status transitions.

CORS Policy

Configurable ALLOWED_ORIGINS. In production must not be localhost (validated by env schema). Credentials enabled for cookie-based auth.

8. Background Jobs (BullMQ + Redis)

When Redis is available, four scheduled jobs run automatically. When Redis is absent (local dev without Docker), the app boots without jobs — all other features work normally.

Hold Expiry every 1 minute

Scans for holds older than 15 minutes and releases the dates. Prevents abandoned holds from blocking bookings indefinitely.

Balance Due Transition every 15 minutes

Moves CONFIRMED_DEPOSIT bookings to BALANCE_DUE when 48h before check-in. Also auto-cancels bookings where balance was not paid within the grace period. Sends reminder emails/SMS.

Payout Eligibility every 1 hour

Marks payout lines as ELIGIBLE 24 hours after guest check-in. This delay allows time for dispute resolution before host is paid.

Weekly Payout Batch Monday 09:00 IST

Aggregates all ELIGIBLE payout lines per host, creates a PayoutBatch record, and transitions lines to SCHEDULED. Admin marks batches as PAID after bank transfer.

11. Guest Hub — Completed Phases

The Guest Hub is the wellness-specific layer on top of the core booking platform. Each phase adds features that go beyond standard booking platforms.

Phase 1 — Wellness Preferences & Retreat Preparation ✅

  • Guest preference form: dietary needs (vegetarian, vegan, gluten-free, ayurvedic), wellness interests (yoga, meditation, Ayurveda, detox), accessibility requirements, room preference, experience level, emergency contact
  • Host-authored preparation guide per listing: packing list, what to expect, daily schedule, dietary information, arrival instructions, additional notes
  • Preparation guide shown to guests only after booking is confirmed

Phase 2 — Guest–Host Messaging ✅

  • Start a conversation from the listing detail page or booking page
  • Threaded message view with send/receive bubbles and auto-scroll
  • Unread count badge on navbar; 60s polling on conversation list, 15s on open thread
  • Host↔Admin messaging channel for operations support

Phase 3 — Guest Assistance Hub ✅

  • Property directions: GPS coordinates, landmarks, transport options, nearest airport/station, parking info
  • Digital property manual: WiFi passwords, house rules, appliance guides, emergency contacts, local tips
  • Issue reporting: 4 categories (Maintenance, Cleanliness, Noise, Safety), 4 urgency levels, photo URL, status tracking (Open → In Progress → Resolved → Closed)
  • Digital check-in: confirm name, estimated arrival, special notes — triggers host notification
  • Digital check-out: feedback and condition notes — triggers host notification and review prompt

12. Roadmap — Future Phases

These phases are defined in the product specification and are ready for development. Each builds on Phase 1's foundation.

Phase 4 Encrypted Access Code Delivery

  • Timed release of door codes, gate codes, and WiFi passwords
  • Encrypted at rest; decrypted only within the check-in time window
  • Prevents early access and automates key handover

Phase 5 Referral System & Loyalty Tiers

  • Guest referral codes with credit rewards for both referrer and referred
  • Loyalty tiers: Seeker (0–2), Practitioner (3–5), Sage (6+) based on completed bookings
  • Platform fee discounts per loyalty tier applied at quote time
  • Credit ledger system for tracking earned and spent credits

Phase 6 Help Center & Price Alerts

  • Self-service FAQ: Booking, Payments, Cancellations, Preparation, Account sections
  • Contact form with booking reference — routes to support team email
  • Price drop alerts: guest sets a target price for a listing → notified when rate drops
  • Automated alert triggering when host updates listing rates

Future Advanced Features (Phase 7+)

  • Advanced Search & Discovery (experience-based filtering, map clusters, property types)
  • Add-on Marketplace (spa sessions, yoga classes, excursions, meals)
  • AI Itinerary Planner based on guest wellness preferences
  • Mobile App (React Native iOS + Android)
  • Group Planning & Expense Splitting
  • International Scaling (multi-currency, multi-language)
  • Smart Property Automation (IoT integration)
  • SOS & Emergency features
  • Trip Savings (SIP-style advance booking)
  • Investor Dashboard & BI Analytics
